# Tourism Service API

Сервис для управления туристическими турами, гидами, направлениями и бронированиями.

##  Основные сущности:
- **Guide** - Гиды с контактной информацией и биографией
- **Destination** - Направления (страны, города)
- **Tour** - Туры с датами, ценой и гидом
- **Booking** - Бронирования туров клиентами
- **Review** - Отзывы о завершенных турах

##  Реализованный функционал:

### CRUD операции:
- ✅ Полное управление гидами (`/api/guides`)
- ✅ Полное управление направлениями (`/api/destinations`)
- ✅ Полное управление турами (`/api/tours`)
- ✅ Полное управление бронированиями (`/api/bookings`)
- ✅ Полное управление отзывами (`/api/reviews`)

### Бизнес-операции:
- ✅ Поиск доступных туров (`GET /api/tours/available`)
- ✅ Проверка доступности конкретного тура (`GET /api/tours/{id}/availability`)
- ✅ Расчет среднего рейтинга тура (`GET /api/reviews/tour/{id}/average-rating`)
- ✅ Поиск бронирований по email клиента (`GET /api/bookings/client/{email}`)
- ✅ Получение отзывов по туру (`GET /api/reviews/tour/{tourId}`)
- ✅ Валидация при создании отзывов (только участники завершенных туров)

##  База данных:
- **PostgreSQL** - реляционная база данных
- **JPA/Hibernate** - ORM маппинг
- **Автоматическое создание схемы** при запуске
- **Тестовые данные** создаются автоматически

##  Связи между сущностями:
- Гид → Туры (One-to-Many)
- Тур → Направления (Many-to-Many)
- Тур → Бронирования (One-to-Many)
- Тур → Отзывы (One-to-Many)
- Бронирование → Тур (Many-to-One)
- Отзыв → Тур (Many-to-One)
